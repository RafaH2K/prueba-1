los fundamentos de la construcion de software incluyen los siguientes:
minimizar la complejidad {
    las personad no pueden recordar facilmente informacion complicada por mucho tiempo, por eso, 
    uno de los objetivos principales al crear software es reducir la complejidad.
    Para lograr reducir la complejidad se usan estandares, dise√±o modular, buenas practicas de codificacion y 
    tecnicas de control de calidad.
    
}
anticipar y acepatar el cambio {
    El software cambia con el tiempo, y por eso, los ingenieros deben de anticipar y aceptar el cambio, esto ayuda a crear 
    programas que sean faciles de mejorar sin tener que cambiar toda su estructura.
    Hoy en dia las empresas necesitan que el software se entregue y actualice mas rapiudo y con mas frecuencia. como es dificil 
    saber exactamente que cambios vendran, los ingenieros deben de hacer que el software sea flexible y adaptable para facilitar 
    futuras modificaciones.
    Para adaptarse mejor a los cambios, los equipos de software usan desarrollo agil, DevOps y practicas de entrega continua,
    que permite que el software evolucione de manera rapuida y ordenada}

construir la verificacion {
    Significa crear el software de forma que sea mas facil encontrar errores durante las pruebas y cuando se este usando.
    para lograrlo se usan varias tecnicas:
    1 Seguir estandares de codificacion para facilitar las revisiones y pruebas.
    2 Organizar el codigo de manera que sea facil de probar automaticacmente.
    3 Evitar usar estructuras de codigo complicadas o dificiles de entender.
    4 Registrar el comportamiento del software usando registros (logs).
}
Reutilizacion de assets{
    La reutilizacion tiene 2 formas:
    1 construccion para la reutilizacion: Crear software que otros puedan reutilizar.
    2. construccion con reutilizacion: Usar software ya hecho para crear algo nuevo.
}
